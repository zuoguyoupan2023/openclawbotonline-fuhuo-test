
问题：
1, 当前项目是一个cloudflare worker项目，部署容器环境然后安装clawdbot；
2, 现在clawdbot的作者因为各种原因把clawdbot改名重新发布了openclaw。
3, dockerfile似乎是用来安装这个旧的clawdbot的，我想知道，现在想改用新的openclaw，需要做哪些修改，其他文件需要修改吗？ 4, openclaw clawdbot的npm页面如下： `https://www.npmjs.com/package/openclaw` `https://www.npmjs.com/package/clawdbot` `/Users/burenweiye/Documents/GitHub/openclawbotonline01/Dockerfile` 
必要的时候可以去了解这两个package更多情况，或者安装到当前项目文件夹/reference里面也行

解决方案：
已完成修改

- Dockerfile 改为安装 openclaw 2026.2.1，并校验 openclaw 版本，仍保留原有目录结构以兼容历史数据： Dockerfile
- 启动脚本改为优先使用 openclaw CLI，若不存在则回退 clawdbot，并建立 /root/.openclaw 与 /root/.openclaw-templates 的软链接以兼容旧路径： start-moltbot.sh
- Worker 的 CLI 调用与调试命令改为 openclaw，同时进程识别支持 openclaw/clawdbot 两种命令名： api.ts , debug.ts , process.ts
- 测试用例更新为 openclaw 命令名： process.test.ts
仍保留 clawdbot 路径的原因

- 当前配置文件与持久化目录仍是 /root/.clawdbot 与 /data/moltbot，以兼容旧数据与 R2 备份结构
- 我已在启动脚本中对 /root/.openclaw 做了软链接，便于新命名的工具读取
你还需要关心的点

- 若你希望彻底迁移目录命名（如 /root/.openclaw、R2 下的 openclaw 目录结构），需要进一步改动：同步脚本、备份路径、Admin 页面显示路径、以及 Debug 读配置路径等
- 目前保留旧路径是最稳妥的“无迁移风险”方案
